# Getaddrinfo

In programmazione C, le funzioni **`getaddrinfo()`** e **`getnameinfo()`** convertono nomi di dominio, nomi di host e indirizzi IP tra rappresentazioni di testo leggibili dall'uomo e formati binari strutturati per l'API di rete del sistema operativo. Entrambe le funzioni sono contenute nell'interfaccia di programmazione dell'applicazione (API) standard POSIX.

getaddrinfo e getnameinfo sono funzioni inverse l'una dell'altra. Sono indipendenti dal protocollo di rete e supportano sia IPv4 che IPv6. È l'interfaccia consigliata per la risoluzione dei nomi nella costruzione di applicazioni indipendenti dal protocollo e per la transizione del codice IPv4 legacy verso Internet IPv6.

Internamente, le funzioni possono utilizzare una varietà di metodi di risoluzione non limitati al Domain Name System (DNS). Il Name Service Switch è comunemente utilizzato su sistemi simili a Unix e influenza la maggior parte delle implementazioni di questa coppia di funzioni, come faceva con i loro predecessori dell'era dei socket BSD.

## addrinfo

La struttura dati C utilizzata per rappresentare i risultati delle query di informazioni di indirizzo per l'uso nell'API di rete potrebbe essere definita come segue, possibilmente con membri in un ordine diverso o membri aggiuntivi:

```
struct addrinfo {
    int ai_flags;
    int ai_family;
    int ai_socktype;
    int ai_protocol;
    socklen_t ai_addrlen;
    struct sockaddr *ai_addr;
    char *ai_canonname;
    struct addrinfo *ai_next;
};
```

La struttura contiene un membro `ai_family` e un membro `ai_addr` che punta a un oggetto `struct sockaddr` con il suo campo `sa_family`. Questi sono impostati allo stesso valore quando la struttura viene creata con la funzione `getaddrinfo()` in alcune implementazioni.

## getaddrinfo

`getaddrinfo()` converte stringhe di testo leggibili dall'uomo che rappresentano nomi di host o indirizzi IP in un elenco di strutture struct addrinfo.
Il prototipo della funzione è:

```
int getaddrinfo(const char *restrict hostname,
       const char *restrict service,
       const struct addrinfo *restrict hints,
       struct addrinfo **restrict res);
```

*hostname*
:   può essere un nome di dominio, come "example.com", una stringa di indirizzo, come "*127.0.0.1*", oppure `NULL`, nel qual caso l'indirizzo *0.0.0.0* o *127.0.0.1* viene assegnato a seconda dei flag di hint.

*service*
:   può essere un numero di porta passato come stringa, come "80", oppure un nome di servizio, ad es. "echo". In quest'ultimo caso, un'implementazione tipica utilizza `getservbyname()` per interrogare il file /etc/services al fine di risolvere il servizio a un numero di porta.

*hints*
:   può essere `NULL` oppure una struttura `addrinfo` con il tipo di servizio richiesto.

*res*
:   è un puntatore che verrà assegnato per puntare a una nuova lista collegata allocata dinamicamente di strutture `addrinfo` con le informazioni richieste dopo il completamento con successo della funzione, da liberare con `freeaddrinfo()`.

La funzione restituisce 0 in caso di successo e un valore di errore diverso da zero se fallisce, che può essere mappato a una stringa leggibile dall'uomo utilizzando `gai_strerror()`

Le implementazioni variano tra le piattaforme, ma spesso la funzione cercherà i nomi host nel DNS, e cercherà i servizi tra un database di servizi ben noti. L'esatto insieme di fonti interrogate per la risoluzione del nome host e del servizio è spesso controllato dalla configurazione del Name Service Switch del sistema.

## freeaddrinfo

La funzione `freeaddrinfo()` libera la memoria allocata dalla funzione `getaddrinfo()`. Data una lista collegata di oggetti `struct addrinfo` a partire da *ai*, restituita da `getaddrinfo(..., &ai)`, `freeaddrinfo(ai)` libera ogni oggetto `struct addrinfo` in quella lista.

```
void freeaddrinfo(struct addrinfo *ai);
```

## getnameinfo

La funzione `getnameinfo()` converte la rappresentazione binaria interna di un indirizzo IP sotto forma di puntatore a una `struct sockaddr` in stringhe di testo costituite dal nome host o, se l'indirizzo non può essere risolto in un nome, una rappresentazione testuale dell'indirizzo IP, così come il nome o il numero della porta di servizio. Il prototipo della funzione è specificato come segue:

```
int getnameinfo(const struct sockaddr *restrict sa, socklen_t salen,
       char *restrict host, socklen_t hostlen,
       char *restrict service, socklen_t servicelen,
       int flags);
```

## Esempio

L'esempio seguente utilizza `getaddrinfo()` per risolvere il nome di dominio www.example.com nella sua lista di indirizzi e quindi chiama `getnameinfo()` su ogni risultato per restituire il nome canonico per l'indirizzo. In generale, questo produce il nome host originale, a meno che l'indirizzo particolare abbia più nomi, nel qual caso viene restituito il nome *canonico*. In questo esempio, il nome di dominio viene stampato tre volte, una per ogni tre risultati ottenuti.

```
#include <errno.h>
#include <stdio.h>
#include <stdlib.h>
#include <string.h>

#include <netdb.h>
#include <netinet/in.h>
#include <sys/socket.h>

#ifndef NI_MAXHOST
#define NI_MAXHOST 1025
#endif

int main(void) {
    const char query_host[] = "www.example.com";
    struct addrinfo *result;

    struct addrinfo hints = {
        .ai_family = AF_UNSPEC,
        .ai_socktype = SOCK_STREAM
    };

    // Resolve the domain name into a list of addresses
    int gai_rc = getaddrinfo(query_host, NULL, &hints, &result);
    switch (gai_rc) {
        case 0:
            break;
        case EAI_SYSTEM:
            fprintf(stderr, "Failed to resolve server address '%s': system error %s\n",
                query_host, strerror(errno));
            return EXIT_FAILURE;
        default:
            fprintf(stderr, "Failed to resolve server address '%s': %s\n",
                query_host, gai_strerror(gai_rc));
            return EXIT_FAILURE;
    }

    // Perform reverse DNS lookup for each resolved address
    for (const struct addrinfo *ai = result; ai != NULL; ai = ai->ai_next) {
        char hostname[NI_MAXHOST];
        char addr[NI_MAXHOST];

        int gni_rc = getnameinfo(ai->ai_addr, ai->ai_addrlen, addr,
            sizeof(addr), NULL, 0, NI_NUMERICHOST);
        if (gni_rc != 0) {
            fprintf(stderr, "Failed to format address: %s\n", gai_strerror(gni_rc));
            continue;
        }

        gni_rc = getnameinfo(ai->ai_addr, ai->ai_addrlen,
            hostname, sizeof(hostname), NULL, 0, 0);
        switch (gni_rc) {
            case 0:
                printf("Hostname for %s: %s\n", addr, hostname);
                break;
            case EAI_SYSTEM:
                fprintf(stderr, "Reverse DNS lookup for %s: system error %s\n",
                    addr, strerror(errno));
                break;
            default:
                fprintf(stderr, "Reverse DNS lookup failed for %s: %s\n",
                    addr, gai_strerror(gni_rc));
        }
    }

    freeaddrinfo(result);
    return 0;
}
```
